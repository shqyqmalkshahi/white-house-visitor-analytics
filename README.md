# white-house-visitor-analytics
Analyzed White House visitor logs to identify patterns in visit volume, location usage, and timing to support operational planning and resource allocation.

# White House Visitor Logs Analysis

##  Overview
This project analyzes historical White House visitor logs to identify patterns in visitor volume, space utilization, and timing.

---

##  Objective
To support operational planning by identifying peak visitation periods, high-usage locations, and demand patterns.

---

# Dataset Information

## Source
White House Visitor Logs (public government dataset)

## Description
This project uses multiple monthly visitor log datasets combined into a single dataset (~392,000 records).

## Access
Due to file size limitations, the dataset is not included in this repository.

You can access the data from:
https://open.whitehouse.gov

## Notes
- Data was cleaned, merged, and standardized across multiple files
- Datetime fields were parsed and transformed for analysis
- No personally identifiable information is used in this project

---

##  Methods
- Data cleaning and standardization across multiple files
- Datetime parsing and feature engineering
- Aggregation and time-based analysis

---

##  Visualizations
- Daily visit volume trends
- Monthly visits by meeting location
- Top meeting rooms by usage
- Visit size over time
- Cumulative visits
- Visits by day of week

---

##  Key Insights
- Visitor demand is uneven and occurs in spikes
- Peak activity occurs toward the end of the week
- A small number of rooms handle most visits
- Large group visits create potential operational strain

---

##  Business Value
- Improves staffing allocation
- Optimizes room scheduling
- Supports data-driven operational planning
